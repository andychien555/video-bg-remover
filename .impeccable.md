# Impeccable — Project Design Context

## Design Context

### Users
影片創作者、設計師、開發者，需要把影片主體去背、輸出「透明背景」的動態素材（動態 WebP / 透明 WebM / PNG 序列），用在貼圖、UI 動畫、直播覆蓋層等。整個運算在瀏覽器內完成（libwebp WASM），不上傳、不留浮水印。使用情境多半是專注、需要精準判斷去背邊緣的「工作台」場景。

### Brand Personality
精準（precise）· 機械感（mechanical）· 沉著自信（quietly confident）。
像一台儀器，不是一個玩具。語氣冷靜、克制、可信賴；少即是多。

### Aesthetic Direction
**Soft Mono — 技術極簡（深色底）**。參考 Linear / Vercel / After Effects 工作台。
- 主題：深色（去背棋盤格、影片畫面在深色下對比最佳，符合工具型產品的使用情境）。
- 色彩：嚴格黑白灰階。中性色微微帶冷藍 hue（cohesion），不引入彩色強調色。層級靠「填滿 vs 描邊 vs 字重 vs 透明度」表達，而非顏色。
- 細節：髮絲線邊框（1px 低透明度）、細網格/點狀分隔、小圓角（6–8px）、克制留白。
- 字體（全非襯線 + mono 點綴）：
  - UI / 標題：**Archivo**（grotesque，精準有重量）。
  - 技術標籤 / 數值 / 程式碼：**Spline Sans Mono**（大寫 + 字距，當作儀器面板標籤）。
  - 中文：系統 PingFang TC / Noto Sans TC fallback。
- Anti-references：iOS 彩色按鈕（紫/綠/橘/洋紅）、漸層、發光、glassmorphism、cyan-on-dark 的 AI 風。

### Design Principles
1. **儀器感優先**：每個控制項看起來都該像精密設備上的旋鈕/讀數，而非消費級 App 按鈕。
2. **黑白灰，靠結構分層級**：禁用彩色強調；用 solid/outline/ghost、字重、透明度建立 hierarchy。唯一的「顏色」是去背棋盤格與影片內容本身。
3. **mono 標籤是點綴不是全部**：英文 mono 大寫只用在標籤/數值，正文與說明用 grotesque 與中文。
4. **髮絲線與留白**：用 1px 低透明邊框與一致的 4pt 間距節奏建立秩序，而非卡片堆卡片。
5. **狀態誠實**：載入 / 處理中 / 就緒 / 錯誤每個狀態都明確，進度可讀。

### Hard Constraints (technical)
- 純 HTML/CSS/JS，無框架、無 build。
- JS 依賴的 DOM `id` 全部不可更名（見 js/app.js 的 `el` 物件）。
- class 契約：`.tab`(`.active`,`data-mode`)、`.preset`(`.active`,`data-color`)。拖曳狀態改用 `.upload-area.dragover`。
